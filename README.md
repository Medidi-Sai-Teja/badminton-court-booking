🏸 Badminton Court Booking System
📌 Project Overview

A full-stack web application designed for college students to book badminton court slots without timetable conflicts. The system ensures smooth booking, prevents double reservations, and keeps users informed with automated notifications.

🚀 Features

✅ User Authentication – Secure login and booking access.

✅ Timetable Validation – Cross-checks student class hours before allowing bookings.

✅ Conflict Prevention – Real-time validation to avoid double-bookings.

✅ Automated Notifications – Sends booking approval/rejection emails.

✅ Admin Dashboard – Manage slots, view booking history, and monitor usage.

🛠️ Tech Stack

Frontend: React.js, CSS, HTML, JavaScript

Backend: Node.js, Express.js

Database: MongoDB

Tools: Git, GitHub, Nodemailer

🧩 Challenges & Solutions

Timetable Conflicts: Integrated validation logic to cross-check schedules.

Double Booking: Real-time database queries ensured unique slot allocation.

Notification Handling: Automated backend email triggers kept users updated.

📊 Impact

Reduced booking conflicts by 90%, improving efficiency and user satisfaction.
